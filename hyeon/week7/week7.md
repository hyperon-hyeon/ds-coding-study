## 1. 놀이기구 1 - 백준 14434번 (런타임 에러)
    

### ❓문제 요약
- **목표**: 놀이기구의 키 제한 조건을 넘어 아이들이 놀이기구를 몇번 타는지 구하기
- **자료구조**: 이분탐색
- **입력**
    - 첫째 줄: 아이들의 수 N, 놀이기구의 수 M, 기간 K, 질문의 개수 Q
    - 둘째 줄: 놀이기구들의 키 제한
    - 셋째 줄: 각 날에 키가 자라는 아이의 번호
    - 넷째 줄부터: Q줄에 걸쳐 (i, j, k) 쌍
- **제약 조건**
    - 모든 놀이기구의 정원은 2명
        - (아이 i의 키) + (아이 j의 키) ≥ (놀이기구 k의 키제한)
        - 단, 놀이기구를 타는 것 보다 키가 자라는 것이 먼저 일어난다
    - 1 ≤ N, M, K, Q ≤ 200,000
    - 1 ≤ 키제한 ≤ 200,000
    - 1 ≤ 아이들의 번호 ≤ N
    - 1 ≤ i, j ≤ N, 1 ≤ k ≤ M
- **출력**
    - k줄에 걸쳐 각 날마다 아이들이 총 몇번씩 놀이기구를 타는지 출력

### ❗코드 설명

**변수**

- growthDays 배열: i번째 아이가 키가 자란 날을 리스트로 저장
- limits 배열: k번 놀이기구의 최소 키 제한 저장
- differenceArray 배열: 일별 탑승 횟수 변화 기록

**코드 설명**

1.  K/2를 선택하여 아이들의 키 합이 놀이기구 제한을 넘는지 확인
    - 날짜가 주어지면 해당 아이의 키가 자란 날짜 계산해서 해당 날짜의 키 계산(이분 탐색 적용)
2. 범위에 따라 최초 탑승일 검색 다시 시도 (재귀적)
    - 넘으면 탑승 가능, 최초 탑승일 범위를 앞쪽으로 줄임
    - 못 넘으면 탑승 불가능, 최초 탑승일 범위를 뒤로 줄임
3. 최초 탑승일을 찾았다면, 해당 일자부터 K일까지 탑승횟수에 1을 더함

## 2. 공유기 설치 - 백준 2110번
### ❓문제 요약

- **목표**: C개의 공유기를 N개의 집에 적당히 설치해서, 가장 인접한 두 공유기 사이의 거리를 최대로 하
- **자료구조**: 이분탐색
- **입력**
    - 첫째 줄: 집의 개수 N, 공유기의 개수 C
    - 둘째 줄부터: N개의 줄에 집의 좌표를 나타내는  X가 한 줄에 하나씩 주어짐
- **제약 조건**
    - 한 집에는 공유기를 하나만 설치
    - 2 ≤ N ≤ 200,000
    - 2 ≤ C ≤ N
    - 0 ≤ xi ≤ 1,000,000,000
- **출력**
    - 첫째 줄에 가장 인접한 두 공유기 사이의 최대 거리를 출력

### ❗코드 설명

- houses: 집의 좌표 저장 배열

**코드 설명**

1. N, C 입력 받고 houses 배열에 저장
2. houses 배열  정렬
3. 설치 가능 여부를 검색
    - 주어진 최소거리를 보장하면서 C개 이상의 공유기를 설치할 수 있는지 확인 (그리디 알고리즘)
        - 첫번째 집에 무조건 공유기 설치, D거리 이상 떨어져 있다면 설치
4. 최소 거리인지 확인하기
    - 설치 가능하다면 더 큰 거리로 설치 가능한지 확인(더 큰 범위 탐색)
    - 설치 불가능하다면 더 좁은 거리로 설치 가능한지 확인 (더 좁은 범위 탐색)
