## 1. 회의실 배정 - 백준 1931번
    

### ❓문제 요약

- **목표**: 한 개의 회의실이 있는데 이를 사용하고자 하는 N개의 회의에 대하여 회의실 사용표 만들기
- **자료구조**: 그리디 알고리즘
- **입력**
    - 첫째 줄: 회의 수 n
    - 둘째 줄부터: 각 회의 정보(회의의 시작 시간, 종료 시간)
    - 셋째 줄부터: 버스의 정보 (버스의 시작 도시 a, 도착 도시 b, 한 번 타는데 필요한 비용 c )
- **제약 조건**
    - 회의의 수는 1 ≤ N ≤ 100,000
    - 시작 시간과 끝나는 시간은 231-1보다 작거나 같은 자연수 또는 0
    - 회의는 한번 시작하면 중간에 중단될 수 없다
    - 한 회의가 끝나는 것과 동시에 다음 회의가 시작될 수 있다
    - 회의의 시작시간과 끝나는 시간이 같을 수도 있다. (시작하자마자 끝남)
- **출력**
    - 최대 사용할 수 있는 회의의 최대 개수를 출력

### ❗코드 설명

**변수**

- N - 회의 수
- time - 입력 받을 회의 시간 저장 배열
    - time[][0] - start / time[][1] - end
- count - 사용 가능한 회의의 최대 개수
- lastEndTime - 회의 끝나는 시간 (계속 갱신)

**코드 설명**

1. 회의 수 만큼 정보를 입력받아 2차원 배열에 저장
2. 종료 시간 기준으로 정렬, 종료 시간 같으면 시작 시간 기준으로 정렬
3. N번만큼 만약 현재 종료 시간보다 시작 시간이 크면 갱신 반복
    - 회의 최대 개수 카운트 하나 증가, 종료 시간 연장
4. 회의 최대 개수 출력

## 2. 크게 만들기 - 백준 2812번
### ❓문제 요약

- **목표**: N자리 숫자가 주어졌을 때, 여기서 숫자 K개를 지워서 얻을 수 있는 가장 큰 수를 구하기
- **자료구조**: 그리디 알고리즘
- **입력**
    - 첫째 줄:  N과 K
    - 둘째 줄부터: N자리 숫자
- **제약 조건**
    - 1 ≤ K < N ≤ 500,000
    - N자리 수는 0으로 시작하지 않는다
- **출력**
    - 입력으로 주어진 숫자에서 K개를 지웠을 때 얻을 수 있는 가장 큰 수를 출력

### ❗코드 설명

**변수**
- N, K: 문제에서 제시한 숫자 자리 수와 지울 개수
- number: 입력 받을 N 자리 숫자 (큰 자리의 숫자라서 String으로 받음)
- result: 문제에 사용될 스택
- deleteCount: 지우고 남은 지울 숫자 (몇자리나 더 지워야하는지)
- currentNum: number 한자리씩 떼어 연산 - 순서대로
- digit: 최종 결과를 합쳐서 출력

**코드 설명**
1. N과 K 숫자 입력 저장
2. N번동안 한 숫자씩 떼어 생각했을 때, 더 큰 숫자가 뒤에 나타났을 때 그보다 작은 수를 제거
    - deleteCount가 0보다 크고 스택이 아직 남아 있고 현재 숫자가 스택 맨 위보다 크면 스택 맨 위 지우고 deleteCount 1 감소
    - 만약 deleteCount 0 되지 못할 경우에 가장 낮은 자리수 제거
3. result에 남은 결과 하나로 합쳐 출력
